# 分析流程架构规范

## 概述

MICOS-2024 采用模块化、容器化的分析流程架构，支持从原始测序数据到最终分析报告的端到端处理。

## 流程阶段

```
┌─────────────────────────────────────────────────────────────────┐
│                    MICOS-2024 分析流程                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Stage 1: 质量控制 (Quality Control)                              │
│   • FastQC: 原始数据质量评估                                      │
│   • KneadData: 适配体去除、宿主过滤、质量修剪                       │
│   输出: clean_reads/{sample}_clean_1.fastq.gz                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Stage 2: 物种分类 (Taxonomic Profiling)                          │
│   • Kraken2: 快速物种分类                                         │
│   • Kraken-BIOM: BIOM 格式转换                                    │
│   • Krona: 交互式可视化                                           │
│   输出: kraken2/{sample}.report, biom/{sample}.biom              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Stage 3: 多样性分析 (Diversity Analysis)                         │
│   • QIIME2: Alpha/Beta 多样性计算                                 │
│   • Phyloseq (R): 统计分析与可视化                                 │
│   输出: diversity/, pcoa/, statistics/                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Stage 4: 功能注释 (Functional Annotation)                        │
│   • HUMAnN3: 基因家族和通路分析                                    │
│   输出: humann/pathways.tsv, humann/gene_families.tsv            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ Stage 5: 结果汇总 (Result Summarization)                         │
│   • HTML 报告生成                                                 │
│   • 交互式图表                                                    │
│   输出: results/report.html, results/figures/                    │
└─────────────────────────────────────────────────────────────────┘
```

## 模块接口规范

### 输入规范

| 阶段 | 输入格式 | 必需参数 |
|------|----------|----------|
| QC | FASTQ (gz) | input_dir, threads |
| 物种分类 | FASTQ (clean) | kraken2_db |
| 多样性 | BIOM | metadata.tsv |
| 功能注释 | FASTQ (clean) | humann_dbs |
| 汇总 | 所有输出 | - |

### 输出规范

```yaml
results/
├── qc/
│   ├── fastqc/
│   └── kneaddata/
├── taxonomic/
│   ├── kraken2/
│   ├── biom/
│   └── krona/
├── diversity/
│   ├── qiime2/
│   └── phyloseq/
├── functional/
│   └── humann/
├── report.html
└── summary.tsv
```

## 并行化策略

- **样本级并行**: 每个样本独立处理
- **阶段级串行**: 阶段间存在依赖关系
- **线程池**: 使用 `concurrent.futures.ThreadPoolExecutor`

## 错误处理

1. 每个阶段独立日志记录
2. 失败样本跳过并记录
3. 最终汇总失败信息

## 扩展机制

新增分析模块需满足：

1. 实现 `BaseModule` 接口
2. 添加 WDL 工作流定义
3. 提供 Dockerfile
4. 编写单元测试
