## 变更类型说明

遵循 [Keep a Changelog](https://keepachangelogs.com/) 格式：

- **Added**: 新增功能
- **Changed**: 功能变更
- **Deprecated**: 即将废弃的功能
- **Removed**: 已移除的功能
- **Fixed**: 问题修复
- **Security**: 安全相关更新

## 如何维护更新日志

### 添加新条目

在 `CHANGELOG.md` 的 `[Unreleased]` 部分添加变更：

```markdown
### Added
- 新功能描述

### Changed
- 变更描述

### Fixed
- 修复描述
```

### 发布新版本

1. 将 `[Unreleased]` 内容移至新版本 section
2. 更新版本号和日期
3. 更新底部链接

### 版本号规则

遵循语义化版本规范：

- **主版本号**: 不兼容的 API 变更
- **次版本号**: 向后兼容的功能新增
- **修订号**: 向后兼容的问题修复

示例：`1.2.3` = `主版本.次版本.修订号`
