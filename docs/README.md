## Documentation Standards

All documentation follows these principles:

- **Task-oriented**: Content organized by what users want to achieve
- **Progressive disclosure**: Simple cases first, advanced topics later
- **Complete examples**: Every command shown with actual parameters
- **Bilingual parity**: English and Chinese versions kept in sync
